# SNS 게시판 프로젝트 정리

> 트위터 스타일의 간단하고 깔끔한 SNS 게시판입니다.
> 제목 없이 본문 중심으로 글을 작성하고, 카테고리, 좋아요, 댓글, 무한 스크롤을 지원합니다.
> 현재는 Vercel에서 UI 미리보기 형태로 배포되어 있으며, 실제 운영은 Oracle 서버에서 영속 DB 기반으로 진행할 예정입니다.

---

## 1. 프로젝트 개요

| 항목       | 내용                                     |
| -------- | -------------------------------------- |
| 프로젝트명    | SNS 게시판                                |
| 프로젝트 목적  | 가볍고 빠른 텍스트 중심 SNS 게시판 구축               |
| 주요 특징    | 제목 없는 글쓰기, 카테고리, 댓글, 좋아요, 무한 스크롤       |

## 2. 프로젝트 목표

이 프로젝트는 복잡한 SNS가 아니라, 다음 목표에 집중합니다.

1. 누구나 쉽게 글을 올릴 수 있는 간단한 게시판
2. 제목 없이 본문만 입력하는 빠른 작성 경험
3. PC와 모바일에서 모두 보기 편한 반응형 화면
4. 좋아요와 댓글을 통한 기본적인 소통 기능
5. Flask와 SQLite만으로 유지보수하기 쉬운 구조
6. Vercel에서는 미리보기, Oracle 서버에서는 실제 운영

## 3. 핵심 컨셉

### 3.1 서비스 컨셉

이 게시판은 트위터나 쓰레드처럼 짧은 글을 빠르게 올리는 구조입니다.

글 작성 시 제목을 입력하지 않고, 현재 선택된 카테고리에 바로 글을 작성합니다.
사용자는 최신 글을 위에서부터 확인하고, 필요하면 댓글을 열어 대화할 수 있습니다.

### 3.2 디자인 컨셉

디자인은 화려함보다 가독성과 속도를 우선합니다.

| 항목     | 기준                   |
| ------ | -------------------- |
| 전체 분위기 | 밝고 단순한 SNS 스타일       |
| 레이아웃   | 중앙 정렬형 피드            |
| 디자인 형태 | 각진 디자인, 라운드 없음       |
| 효과     | 그림자 없음, 과한 애니메이션 없음  |
| 우선순위   | 시인성, 터치 편의성, 모바일 최적화 |
| 입력 방식  | 빠른 글쓰기, 자동 높이 조절     |

## 5.7 무한 스크롤

게시글은 한 번에 전체를 불러오지 않고, 일정 개수씩 추가 로딩합니다.

| 항목    | 설명                    |
| ----- | --------------------- |
| 기본 정렬 | 최신순                   |
| 방식    | offset, limit         |
| 장점    | 초기 로딩 속도 개선           |
| 추가 로딩 | 화면 하단 접근 시 자동 요청      |
| 중복 방지 | loading 상태로 중복 요청 차단  |
| 종료 처리 | 더 이상 불러올 글이 없으면 요청 중단 |

예시 요청:

```http
GET /api/posts?category=free&offset=0&limit=10
```

## 10.4 likes

게시글 좋아요 정보를 저장합니다.

| 컬럼      | 설명     |
| ------- | ------ |
| post_id | 게시글 ID |
| user_id | 사용자 ID |

특징:

```sql
PRIMARY KEY (post_id, user_id)
```

게시글 1개에 대해 사용자 1명은 좋아요를 1번만 누를 수 있습니다.

## 10.5 삭제 관계

외래키는 `ON DELETE CASCADE`를 사용합니다.

| ------ | -------------------- |
| 게시글 삭제 | 댓글, 좋아요              |
| 사용자 삭제 | 해당 사용자의 게시글, 댓글, 좋아요 |

## 25.3 첨부파일 기능 설계안

첨부파일 기능은 게시글 작성 시 파일을 함께 업로드하는 방식으로 확장할 수 있습니다.

### 지원 권장 파일

| 파일 종류 | 확장자                       |
| ----- | ------------------------- |
| 이미지   | jpg, jpeg, png, webp, gif |
| 문서    | pdf                       |
| 일반 파일 | 필요 시 제한적으로 허용             |

### 권장 제한

| 항목        |               기준 |
| --------- | ---------------: |
| 파일 크기     |       파일당 5MB 이하 |
| 게시글당 파일 수 |            3개 이하 |
| 저장 위치     | `uploads/posts/` |
| 파일명       |       UUID 기반 저장 |
| 원본 파일명    |        DB에 별도 저장 |

### 추가 테이블 예시

```sql
CREATE TABLE IF NOT EXISTS attachments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    post_id INTEGER NOT NULL,
    original_name TEXT NOT NULL,
    stored_name TEXT NOT NULL,
    file_path TEXT NOT NULL,
    mime_type TEXT,
    file_size INTEGER,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (post_id) REFERENCES posts(id) ON DELETE CASCADE
);
```

### 첨부파일 API 예시

| 메서드    | 경로                          | 설명               |
| ------ | --------------------------- | ---------------- |
| POST   | `/api/posts`                | 글 작성 시 파일 함께 업로드 |
| GET    | `/uploads/posts/<filename>` | 첨부파일 조회          |
| DELETE | `/api/attachments/<id>`     | 첨부파일 삭제          |

## 29. 개발 기준 요약

이 프로젝트를 수정할 때는 아래 기준을 유지합니다.

1. 제목 없는 본문 중심 SNS 구조 유지
2. 글쓰기 영역에 카테고리 선택창 추가하지 않음
3. 카테고리는 왼쪽 메뉴 기준으로 작성
4. 최신 글이 항상 최상단
5. 모바일 화면에서 가로 스크롤 금지
6. 각진 디자인 유지
7. 그림자와 과한 장식 사용하지 않음
8. 운영 DB는 GitHub에 올리지 않음
9. Vercel은 미리보기 전용으로 사용
10. 실제 운영은 Oracle 서버에서 영속 DB로 관리

## 30. 최종 정리

SNS 게시판은 현재 MVP 수준의 핵심 기능을 갖춘 상태입니다.

회원가입, 로그인, 글 작성, 카테고리, 댓글, 좋아요, 무한 스크롤, 모바일 최적화까지 완료되어 있으며, Vercel을 통해 미리보기 배포도 가능합니다.

다만 Vercel 환경의 SQLite DB는 영속 저장이 불가능하므로, 실제 서비스 운영은 Oracle 서버에서 진행해야 합니다.

다음 단계는 게시글 검색, 첨부파일, 프로필, 관리자 기능을 추가하여 단순 게시판에서 실제 운영 가능한 소형 SNS 서비스로 고도화하는 것입니다.
