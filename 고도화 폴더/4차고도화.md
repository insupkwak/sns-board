# SNS 게시판 디자인 고도화 MD

> 기존 밝은 톤의 단순 게시판 디자인을 버리고, 어두운 톤의 고급스럽고 세련된 대형 SNS 스타일로 전면 개선한다.
> 목표는 Instagram, X, Threads, Discord, 최신 SaaS 대시보드 감각에 뒤처지지 않는 프리미엄 다크 SNS 게시판 UI를 만드는 것이다.

---

# 1. 디자인 고도화 목표

## 1.1 기존 디자인 문제점

현재 디자인은 기능은 단순하고 사용하기 쉽지만, 대형 SNS 서비스처럼 세련된 인상을 주기에는 부족하다.

현재 문제점은 다음과 같다.

| 항목      | 문제                             |
| ------- | ------------------------------ |
| 색감      | 밝은 배경 위주의 기본 게시판 느낌            |
| 분위기     | 개인 프로젝트 또는 테스트 게시판 느낌이 강함      |
| 피드 카드   | 대형 SNS 서비스의 고급스러운 밀도 부족        |
| 탐색 화면   | 관심사, 트렌드, 추천 영역의 시각적 매력 부족     |
| 모바일 UI  | 기능은 되지만 앱 같은 감각 부족             |
| 버튼 스타일  | 단순하고 브랜드 느낌이 약함                |
| 인터랙션    | hover, active, selected 상태가 약함 |
| 브랜드 정체성 | 이 플랫폼만의 색상과 분위기가 부족함           |

---

## 1.2 개선 목표

이번 디자인 개선의 목표는 다음이다.

| 목표           | 설명                               |
| ------------ | -------------------------------- |
| 다크 프리미엄 톤    | 검정, 딥네이비, 차콜 계열 기반               |
| 고급스러운 SNS 감각 | Instagram, X, Threads 느낌의 세련된 피드 |
| 쉬운 탐색        | 검색, 태그, 트렌드, 게시판 찾기를 직관적으로 구성    |
| 모바일 앱 같은 경험  | 하단 탭, 큰 터치 영역, 부드러운 카드 구조        |
| 브랜드감 강화      | 포인트 컬러, 그라디언트, 통일된 컴포넌트 적용       |
| 가독성 유지       | 어두운 배경에서도 글이 잘 읽혀야 함             |
| 과하지 않은 효과    | 그림자, 블러, 애니메이션은 절제해서 사용          |
| 개발 적용 용이     | CSS 변수와 공통 클래스로 일괄 적용 가능해야 함     |

# 3. 디자인 기본 원칙

## 3.1 전체 원칙

1. 전체 배경은 어두운 톤으로 변경한다.
2. 완전한 검정만 쓰지 말고 딥네이비와 차콜을 섞는다.
3. 피드 카드는 배경보다 살짝 밝은 다크 카드로 만든다.
4. 테두리는 강한 흰색이 아니라 은은한 라인으로 처리한다.
5. 버튼은 포인트 컬러와 그라디언트를 적절히 사용한다.
6. 글자는 회색이 아니라 충분히 밝은 색으로 읽기 쉽게 한다.
7. 모바일에서는 하단 탭을 앱처럼 고정한다.
8. 검색창은 서비스의 중심 기능처럼 크게 보여준다.
9. 인기 태그, 트렌드, 추천 게시판은 칩과 카드로 시각화한다.
10. 과한 그림자와 번쩍이는 애니메이션은 금지한다.

---

## 3.2 절대 금지 사항

| 금지 항목           | 이유                 |
| --------------- | ------------------ |
| 밝은 흰색 배경 중심 디자인 | 기존 게시판 느낌이 강함      |
| 너무 많은 색상 사용     | 서비스가 산만해짐          |
| 강한 네온 남발        | 저가 게임 UI처럼 보일 수 있음 |
| 카드마다 다른 스타일     | 일관성 깨짐             |
| 작은 글씨 남발        | 모바일 가독성 저하         |
| 과도한 그림자         | 다크 UI에서 지저분해짐      |
| 둥근 정도가 제각각      | 디자인 완성도 저하         |
| 버튼 색상 무작위 사용    | 브랜드감 약화            |
| 모든 영역에 blur 적용  | 성능 저하와 가독성 저하      |
| 클릭 가능한 요소 구분 부족 | 사용성 저하             |

# 5. 전체 배경 디자인

## 5.1 body 스타일

전체 배경은 단순 검정이 아니라 은은한 그라디언트가 들어간 다크 배경으로 만든다.

```css
html,
body {
  margin: 0;
  min-height: 100%;
  background: var(--grad-bg);
  color: var(--text-main);
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Noto Sans KR", Arial, sans-serif;
  overflow-x: hidden;
}

body {
  color-scheme: dark;
}
```

---

## 5.2 배경 효과

배경은 고급스럽게 보이되, 콘텐츠보다 튀면 안 된다.

적용 기준:

1. 상단 좌측에 블루 빛을 약하게 둔다.
2. 상단 우측에 퍼플 빛을 약하게 둔다.
3. 중앙 피드 영역은 충분히 어둡게 유지한다.
4. 배경 애니메이션은 사용하지 않는다.
5. 모바일에서는 배경 효과를 약하게 줄인다.

# 9. 중앙 피드 디자인

## 9.1 피드 원칙

중앙 피드는 서비스의 핵심이다.

피드는 다음 느낌이어야 한다.

```text
읽기 편함
카드 간격 적당함
콘텐츠 집중
좋아요, 댓글, 공유 버튼이 자연스러움
추천 게시판과 트렌드 카드가 중간에 섞여도 어색하지 않음
```

---

## 9.2 피드 헤더

피드 상단에는 현재 화면명을 표시한다.

```text
홈
오늘 당신의 관심사에서 가장 많이 이야기되는 글
```

CSS:

```css
.feed-header {
  padding: 18px 4px 14px;
}

.feed-title {
  margin: 0;
  font-size: 28px;
  line-height: 1.15;
  font-weight: 850;
  letter-spacing: -0.05em;
}

.feed-subtitle {
  margin-top: 6px;
  color: var(--text-muted);
  font-size: 14px;
}
```

---

## 9.3 피드 카드


# 24. 모바일 디자인 세부 기준

## 24.1 모바일 기본 원칙

1. 좌우 여백은 10px에서 12px로 한다.
2. 피드 카드는 화면 폭을 거의 꽉 채운다.
3. 터치 영역은 최소 44px 이상 유지한다.
4. 하단 탭 때문에 콘텐츠가 가려지지 않도록 padding-bottom을 준다.
5. 검색창은 상단에 크게 배치한다.
6. 글쓰기 버튼은 하단 중앙 또는 피드 상단에 명확히 둔다.
7. 모달은 작은 화면에서 하단 시트 형태도 가능하다.
8. iOS 입력 확대 방지를 위해 input, textarea는 16px 이상으로 한다.

# 32. 개발자에게 줄 최종 명령

아래 명령을 기준으로 디자인 개선 작업을 진행한다.

```text
현재 sns-board 프로젝트의 디자인을 전면 개선한다.

목표는 밝은 게시판 느낌을 제거하고, Instagram, X, Threads 같은 최신 SNS에 뒤처지지 않는 다크 프리미엄 SNS 디자인으로 바꾸는 것이다.

static/css/style.css 상단에 디자인 토큰을 추가하고, 전체 배경, 카드, 버튼, 검색창, 글쓰기 카드, 게시글 카드, 게시판 카드, 트렌드 카드, 모달, 모바일 하단 탭을 모두 이 토큰 기준으로 재정리한다.

templates/index.html은 app-shell 구조로 정리한다.
PC에서는 left-rail, feed, right-rail의 3열 구조를 사용한다.
모바일에서는 1열 피드와 fixed mobile-tabbar를 사용한다.

기존 기능은 절대 깨지지 않게 한다.
회원가입, 로그인, 글쓰기, 카테고리, 좋아요, 댓글, 무한 스크롤은 그대로 유지한다.
디자인만 바꾸되 필요한 클래스는 추가할 수 있다.

디자인 기준은 다음과 같다.

1. 전체 배경은 딥네이비와 블랙 계열의 다크 그라디언트로 한다.
2. 게시글 카드는 다크 카드, 얇은 테두리, 은은한 hover를 적용한다.
3. 글쓰기 영역은 glass 느낌의 프리미엄 카드로 만든다.
4. 버튼은 pill 형태와 브랜드 그라디언트를 사용한다.
5. 검색창은 크고 명확하게 만든다.
6. 관심사와 태그는 pill chip으로 만든다.
7. 오른쪽 패널에는 트렌드와 추천 게시판을 카드로 보여준다.
8. 모바일에는 하단 탭을 추가한다.
9. 모든 클릭 가능한 요소에는 hover, active, focus-visible 상태를 만든다.
10. 320px, 360px, 390px 모바일 화면에서 가로 스크롤이 절대 생기지 않게 한다.
11. input, textarea는 모바일 자동 확대 방지를 위해 16px 이상으로 한다.
12. 로딩 상태 skeleton과 빈 상태 empty-state를 추가한다.
13. 색상, radius, border, shadow는 CSS 변수만 사용해 통일한다.
14. 과한 네온, 과한 blur, 무거운 애니메이션은 사용하지 않는다.
15. 최종 결과는 고급스럽고 빠르고 읽기 쉬운 대형 SNS 게시판처럼 보여야 한다.
```

# 33. 최종 정리

이번 디자인 고도화의 핵심은 단순히 어둡게 바꾸는 것이 아니다.

목표는 다음이다.

```text
기능은 단순하게
화면은 고급스럽게
탐색은 쉽게
피드는 읽기 좋게
모바일은 앱처럼
전체는 대형 SNS처럼
```

이 디자인 기준을 적용하면 기존 `sns-board`는 단순한 Flask 게시판 느낌에서 벗어나, 실제 서비스처럼 보이는 다크 프리미엄 SNS 플랫폼으로 발전할 수 있다.
