# SNS Board 다크 프리미엄 디자인 완성형 고도화 작업 지시서

> 현재 `sns-board` 프로젝트를 대형 SNS 스타일의 다크 프리미엄 플랫폼으로 고도화한다.
> 단순히 CSS만 바꾸는 작업이 아니라, 디자인에 필요한 화면, 버튼, 상태, 검색, 탐색, 트렌드, 모바일 하단 탭, 게시판 카드, 알림 배지, 로딩, 빈 상태까지 실제로 작동하도록 구현한다.

---

# 1. 작업 목표

현재 프로젝트는 다음 기능이 이미 작동한다.

```text
회원가입
로그인
로그아웃
게시글 작성
게시글 수정
게시글 삭제
카테고리
좋아요
댓글
무한 스크롤
모바일 반응형
```

이번 작업의 목표는 기존 기능을 절대 깨지 않고, 아래 기능과 디자인을 완성하는 것이다.

```text
다크 프리미엄 전체 디자인
PC 3열 레이아웃
모바일 하단 탭
검색 인터페이스
탐색 화면
트렌드 패널
게시판 카드
추천 게시판 카드
글쓰기 카드 고급화
게시글 카드 고급화
관심사 칩
로딩 스켈레톤
빈 상태 화면
알림 배지 UI
좋아요, 댓글, 공유 액션 UI
모달 디자인 개선
모바일 앱형 UX
```

---

# 2. 절대 유지해야 할 기존 기능

디자인 고도화 중 아래 기능은 반드시 유지한다.

| 기능        | 유지 조건                                |
| --------- | ------------------------------------ |
| 회원가입      | 기존 `/api/register` 그대로 작동            |
| 로그인       | 기존 `/api/login` 그대로 작동               |
| 로그아웃      | 기존 `/api/logout` 그대로 작동              |
| 현재 사용자 조회 | 기존 `/api/me` 그대로 작동                  |
| 게시글 목록    | 기존 `/api/posts` 그대로 작동               |
| 게시글 작성    | 기존 `/api/posts` POST 그대로 작동          |
| 게시글 수정    | 기존 PUT `/api/posts/<id>` 그대로 작동      |
| 게시글 삭제    | 기존 DELETE `/api/posts/<id>` 그대로 작동   |
| 좋아요       | 기존 `/api/posts/<id>/like` 그대로 작동     |
| 댓글 목록     | 기존 `/api/posts/<id>/comments` 그대로 작동 |
| 댓글 작성     | 기존 댓글 작성 API 그대로 작동                  |
| 댓글 수정     | 기존 댓글 수정 API 그대로 작동                  |
| 댓글 삭제     | 기존 댓글 삭제 API 그대로 작동                  |
| 무한 스크롤    | 기존 offset, limit 방식 유지 가능            |
| 카테고리      | 전체, 자유, 질문, 정보, 일상, 유머 유지            |

---

# 3. 이번 고도화의 핵심 기준

이번 디자인은 다음 기준으로 구현한다.

```text
기능은 기존 그대로 유지한다.
화면 구조는 대형 SNS 스타일로 바꾼다.
PC는 3열 구조로 만든다.
모바일은 1열 피드와 하단 탭으로 만든다.
전체 색상은 다크 프리미엄 톤으로 만든다.
글쓰기, 게시글, 게시판, 트렌드, 검색은 모두 카드형으로 만든다.
검색과 탐색 기능은 실제로 동작하게 만든다.
트렌드와 추천 게시판은 초기에는 서버에서 계산하지 않아도 된다.
프론트에서 기본 데이터 또는 기존 게시글 데이터를 활용해 표시한다.
추후 DB 확장 가능하도록 구조만 미리 잡는다.
```

# 5. 파일별 작업 범위

| 파일                     | 작업 내용                                         |
| ---------------------- | --------------------------------------------- |
| `templates/index.html` | 전체 레이아웃을 app-shell 구조로 재정리                    |
| `static/css/style.css` | 다크 프리미엄 디자인 전체 적용                             |
| `static/js/app.js`     | 화면 전환, 검색, 탐색, 트렌드, 모바일 탭 상태 추가               |
| `app.py`               | 필요한 경우 검색 API와 추천 데이터 API 추가                  |
| `data/data.sql`        | 이번 단계에서는 필수 변경 없음, 선택적으로 boards, trends 확장 가능 |

# 6. 1차 구현 범위

이번 작업에서는 아래 기능을 실제로 작동하게 만든다.

| 구분     | 기능               | 구현 방식                             |
| ------ | ---------------- | --------------------------------- |
| 디자인    | 다크 프리미엄 UI       | CSS 변수 기반                         |
| 레이아웃   | PC 3열 구조         | HTML 구조 변경                        |
| 모바일    | 하단 탭             | CSS fixed nav, JS 화면 전환           |
| 홈      | 기존 피드 표시         | 기존 `/api/posts` 사용                |
| 탐색     | 게시판, 태그, 검색 UI   | 프론트 렌더링                           |
| 검색     | 게시글 본문 검색        | `/api/posts?search=` 추가 또는 프론트 필터 |
| 트렌드    | 인기 태그, 인기글 표시    | 기존 게시글 데이터 기반                     |
| 추천 게시판 | 기본 추천 카드 표시      | 프론트 정적 데이터 가능                     |
| 로딩     | 스켈레톤 표시          | JS 로딩 상태                          |
| 빈 상태   | 게시글 없음 화면        | JS 렌더링                            |
| 모달     | 로그인, 회원가입 디자인 개선 | 기존 기능 유지                          |
| 액션     | 좋아요, 댓글 버튼 상태 개선 | 기존 API 유지                         |

## 40.2 게시글 로딩 시 검색어 반영

```js
async function loadPosts() {
  if (state.loading || !state.hasMore) return;

  state.loading = true;

  const params = new URLSearchParams();
  params.set("offset", state.offset);
  params.set("limit", state.limit);

  if (state.currentCategory && state.currentCategory !== "all") {
    params.set("category", state.currentCategory);
  }

  if (state.searchQuery) {
    params.set("search", state.searchQuery);
  }

  const res = await fetch(`/api/posts?${params.toString()}`);
  const data = await res.json();

  const posts = Array.isArray(data.posts) ? data.posts : data;

  if (state.offset === 0 && posts.length === 0) {
    document.getElementById("postList").innerHTML = "";
    renderEmptyState("검색 결과가 없습니다.", "다른 키워드로 다시 검색해보세요.");
  } else {
    document.getElementById("emptyState").hidden = true;
    appendPosts(posts);
  }

  state.offset += posts.length;
  state.hasMore = posts.length >= state.limit;
  state.loading = false;
}
```

# 42. 유틸 함수

JS에 아래 유틸 함수를 추가한다.

```js
function escapeHtml(value) {
  return String(value ?? "")
    .replaceAll("&", "&amp;")
    .replaceAll("<", "&lt;")
    .replaceAll(">", "&gt;")
    .replaceAll('"', "&quot;")
    .replaceAll("'", "&#039;");
}

function numberFormat(value) {
  return Number(value || 0).toLocaleString("ko-KR");
}

function formatDate(value) {
  if (!value) return "";
  const date = new Date(value);
  if (Number.isNaN(date.getTime())) return value;
  return date.toLocaleString("ko-KR", {
    month: "2-digit",
    day: "2-digit",
    hour: "2-digit",
    minute: "2-digit"
  });
}
```

# 55. 최종 정리

이번 고도화 작업은 단순 디자인 변경이 아니다.

현재 `sns-board`를 다음 단계로 올리는 작업이다.

```text
기존 Flask 게시판
→ 다크 프리미엄 SNS UI
→ 모바일 앱형 하단 탭
→ 탐색과 트렌드
→ 게시판 추천
→ 검색 가능
→ 실제 서비스처럼 보이는 SNS 플랫폼
```

이번 작업이 완료되면 사용자는 처음 접속했을 때 단순 게시판이 아니라, 실제 대형 SNS 서비스에 가까운 완성도를 느끼게 된다.

디자인은 고급스럽고, 기능은 기존처럼 안정적으로 작동하며, 이후 게시판 팔로우, 관심사 추천, 주제방, 알림, 관리자 기능으로 확장할 수 있는 기반이 완성된다.
